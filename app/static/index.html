Recover failed payments
before they become churn.

When a subscription charge fails, the customer didn't decide to leave - their card just expired or a payment bounced. PayPilot turns each failed-payment event into a grounded recovery action: it diagnoses why, picks the right retry strategy, and drafts a warm, one-click dunning email - in a single API call.

How it works

A failed-payment event flows through a typed LangGraph state machine. Each node enriches a shared state and hands it on.

🔎

Retrieve + diagnose

Loads the customer record and pulls relevant snippets from the dunning playbook (RAG), then an LLM writes a short, grounded diagnosis of why the charge failed.

⚙️

Deterministic strategy

The retry cadence and action come from a fixed rules table keyed on the failure code - not an LLM - so the policy stays stable and unit-testable.

✉️

Draft the email

A second LLM call writes a warm, on-brand dunning email with one clear call to action, grounded in the playbook's tone guidance.
